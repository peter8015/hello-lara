
1. 算法题（Coding/Data Structures）：依然是“硬门槛”
   对于大厂（Google, Meta, Amazon 等），算法题通常是第一轮甚至前两轮的“生死线”。
   现状： 不再单纯追求超难的动态规划（DP），而更看重代码质量、沟通能力和边界条件的处理。
   策略： 你不需要刷 1000 题，但需要熟练掌握 Medium 难度的常考题。面试官更想看你在写代码时是否会边写边解释思路（Communication）。
2. 系统设计（System Design）：高级岗位的“重头戏”
   如果你面试的是高级开发或架构师，系统设计的分量往往超过算法。
   考察点： 如何设计一个像 Twitter 这样的系统？如何处理高并发？如何保证数据一致性（Consistency vs Availability）？
   趋势： 现在的面试官非常讨厌背诵模板。他们会深挖细节，比如“为什么要用 Redis 而不是 Memcached？”或者“在这个特定场景下，数据库索引该怎么建？”
3. OOD 与实战（Object-Oriented Design / Practical Coding）
   越来越多的公司开始放弃 LeetCode，转而考察实战能力：
   任务： 给你 1 小时，在一个现有的框架下实现一个功能（比如一个简单的限流器，或者一个 API 接口）。
   重点： 考察你如何组织代码逻辑、是否写 Unit Test、以及代码的可读性（Readability）。
4. 行为面试（Behavioral Interview / BQ）
   在北美，这部分极其重要，甚至有一票否决权。
   核心： 亚马逊的“领导力准则”（Leadership Principles）是其中的典型。
   考察点： 面对冲突你怎么处理？你最失败的一个项目是什么？你如何说服你的 Manager？
   方法： 必须使用 STAR 法则（Situation, Task, Action, Result）来准备你的故事。
