# springboot

## Spring Boot 的自动装配原理是什么？

Spring Boot 的自动装配原理是基于 Spring 框架的 @EnableAutoConfiguration 注解和 @Configuration 注解的组合使用。Spring Boot 通过扫描类路径下的配置类和自动配置类，以及读取 META-INF/spring.factories 文件来实现自动装配。

以下是 Spring Boot 自动装配的源码角度的详细解释：

    1.
    @EnableAutoConfiguration 注解:
        这个注解是自动装配的入口点。它本身并不做任何事情，但它会触发 Spring Boot 的自动配置机制。
        @EnableAutoConfiguration 注解通常与 @SpringBootApplication 注解一起使用，后者是一个组合注解，包含了 @Configuration、@EnableAutoConfiguration 和 @ComponentScan。
    2.
    @Configuration 注解:
        @Configuration 注解标记的类会被 Spring 容器识别为配置类，其中可以包含 @Bean 注解的方法，用于定义和初始化 Spring 容器中的 Bean。
    3.
    META-INF/spring.factories 文件:
        Spring Boot 在其类路径下包含了一个或多个 META-INF/spring.factories 文件，这些文件列出了所有需要自动配置的类。
        这些文件通常位于 Spring Boot 的 starter 依赖中，例如 spring-boot-autoconfigure starter。
        spring.factories 文件中的每一行都是一个键值对，键通常是 EnableAutoConfiguration，值是一个配置类的全类名列表，这些配置类会被自动加载。
    4.
    自动配置类:
        自动配置类通常使用 @Configuration 注解，并且可能包含 @Conditional 注解，这些注解用于指定自动配置类的生效条件。
        自动配置类中可能包含 @Bean 注解的方法，用于定义 Bean，这些 Bean 会被自动注册到 Spring 容器中。
    5.
    条件注解:
        @Conditional 注解用于指定自动配置类的生效条件，例如 @ConditionalOnClass、@ConditionalOnBean、@ConditionalOnMissingBean 等。
        这些条件注解确保只有在特定的类存在、特定的 Bean 存在或缺失时，自动配置类才会生效。
    6.
    自动装配流程:
        当 Spring Boot 应用启动时，会通过 SpringFactoriesLoader 类加载 META-INF/spring.factories 文件中的配置。
        SpringFactoriesLoader 会读取文件中的键值对，并实例化所有指定的自动配置类。
        自动配置类中的 @Bean 方法会被调用，从而创建 Bean 实例并注册到 Spring 容器中。
        最终，Spring 容器中包含了所有通过自动配置创建的 Bean，实现了自动装配。

通过以上步骤，Spring Boot 能够自动识别和配置应用程序所需的 Bean，大大简化了 Spring 应用的配置过程。开发者只需要添加相应的 starter 依赖，Spring Boot 就会自动处理剩下的配置工作。

# BeanFactory` 和 `FactoryBean区别

`BeanFactory` 和 `FactoryBean` 是Spring框架中两个不同的概念，它们在Spring容器中扮演着不同的角色。

### BeanFactory

`BeanFactory` 是Spring容器的核心接口，它负责管理Spring容器中所有的Bean。`BeanFactory` 提供了基本的依赖注入功能，包括创建Bean实例、管理Bean的生命周期、依赖注入等。`BeanFactory` 是Spring容器的根接口，它定义了Spring容器的基本行为。

### FactoryBean

`FactoryBean` 是一个特殊的Bean，它实现了`FactoryBean`接口。当Spring容器需要创建这个Bean的实例时，它不是直接创建`FactoryBean`本身，而是调用`FactoryBean`的`getObject()`方法来获取实际的Bean实例。`FactoryBean`通常用于创建复杂的Bean，这些Bean的创建过程可能涉及多个步骤或依赖于外部资源。
